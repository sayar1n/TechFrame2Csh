# Мини веб-служба и конвейер обработки запросов (модули + DI)

## Что сделано

Реализовано приложение на Node.js, расширяемое модулями без переписывания ядра:

- У каждого модуля есть **имя**, **зависимости**, и два шага: **`register(container)`** и **`init(container)`**.
- Приложение читает список модулей из `config/modules.json` и загружает их из каталога `modules/`.
- Порядок запуска строится с учётом зависимостей (топологическая сортировка).
- Если модуль отсутствует или зависимость не удовлетворена — приложение останавливается с понятной ошибкой.
- Если есть цикл зависимостей — приложение останавливается с понятной ошибкой.
- Модули регистрируют службы через DI-контейнер (`src/container.js`).
- После запуска поведение реально меняется за счёт модулей конвейера обработки запросов.

## Структура

- `src/main.js` — запуск приложения: загрузка модулей, построение порядка, `register`, затем `init`, затем запуск действий.
- `src/moduleLoader.js` — загрузка модулей из конфига и построение порядка с проверками.
- `src/container.js` — простой DI-контейнер (singleton/transient).
- `src/httpPipeline.js` — компоновка конвейера (`middleware.*`).
- `modules/` — модули расширения.
- `config/modules.json` — список модулей (файлы из `modules/`).
- `test/` — проверки требований.

## Контракт модуля

Каждый модуль экспортирует `default` объект:

- `name: string`
- `requires: string[]`
- `register(container)`
- `init(container)` (может быть async)

## Модули (пример поведения)

- `Core`:
  - Регистрирует `http.server` и запускает сервер на `:3000`
  - Собирает конвейер из всех `middleware.*`
- `Logging`:
  - Регистрирует `logger`
  - Добавляет middleware логирования начала/конца запроса
- `Validation`:
  - Проверяет `/process?name=...`
  - Если `name` пуст — отвечает `400` и завершает обработку
- `Export`:
  - После успешной обработки `/process` дописывает строку в `export.txt`
- `Report`:
  - Ведёт счётчик обработок `/process` в singleton-службе `stats`

## Запуск

Установки зависимостей не требуется (используются только встроенные модули Node.js).

### Запуск приложения

```bash
npm start
```

После старта:

- Откройте `http://localhost:3000/process?name=Иван`
- Откройте `http://localhost:3000/process` (получите 400 с понятной ошибкой)
- Проверьте, что появился/пополняется файл `export.txt` в текущей папке запуска

## Проверки (требования к испытаниям)

Запуск тестов:

```bash
npm test
```

Покрыто:

- Проверки порядка запуска на нескольких наборах зависимостей: `test/moduleLoader.test.js`
- Проверка ошибки отсутствующего модуля с понятным сообщением: `test/moduleLoader.test.js`
- Проверка ошибки циклических зависимостей с понятным сообщением: `test/moduleLoader.test.js`
- Проверка, что зависимости внедряются через контейнер: `test/di.test.js`

## Что относится к ядру, а что к модулям

- Ядро: `src/` (загрузка, порядок, контейнер, конвейер).
- Модули: `modules/` (логика поведения и регистрация служб).

Добавление нового модуля не требует правок логики запуска: достаточно добавить файл в `modules/` и указать его в `config/modules.json`.
